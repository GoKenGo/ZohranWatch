First off, solidarity! Thank you for considering contributing to Zohran Watch. This is a community-built accountability tool, and we welcome contributions from organizers, activists, and community members.

## üìã How to Contribute

### Reporting Promise Status Updates

If you have evidence that a promise status should be updated:

1. **Open an Issue** with the title: `[Status Update] Promise Name`
2. **Provide Evidence:**
   - Link to official announcements, legislation, or news articles
   - Date of the update
   - Proposed new status (Delivered, In Progress, Blocked, etc.)
3. **Explain the Change:**
   - What happened?
   - Why does this warrant a status change?

**Example:**
```
Title: [Status Update] Rent Freeze

The Rent Guidelines Board voted 5-4 to implement a citywide rent freeze on February 28, 2026.

Evidence:
- Official RGB announcement: [link]
- NYC.gov press release: [link]
- Coverage in Gothamist: [link]

Proposed Status: Delivered
Reason: The promise has been fully implemented as of the RGB vote.
```

### Adding News Updates

To add a news item to the tracker:

1. **Fork the repository**
2. **Edit `index.html`**
3. **Add a news item** to the `#newsFeed` section:

```html
<div class="news-item">
    <div class="news-date">March 15, 2026</div>
    <div class="news-title">Your News Title Here</div>
    <div class="news-content">Your description here, with links to sources.</div>
</div>
```

4. **Submit a Pull Request** with a clear description

### Reporting Bugs

Found a bug? Please open an issue with:

- **Clear description** of the problem
- **Steps to reproduce** the bug
- **Expected behavior** vs actual behavior
- **Screenshots** if applicable
- **Browser/device** information

### Design Improvements

We love design contributions that maintain the protest poster aesthetic!

**Design Principles to Follow:**
- NYC labor movement poster energy
- Risograph zine textures and colors
- Hand-crafted, grassroots feel
- Red/Blue/Cream/Black color palette
- Bebas Neue + Courier Prime fonts

**Before submitting:**
1. Open an issue describing your proposed changes
2. Get feedback from maintainers
3. Submit a PR with before/after screenshots

### Code Contributions

**Technical Stack:**
- Pure HTML/CSS/JavaScript (no frameworks)
- Twitter Widgets for feed integration
- Google Fonts (Bebas Neue, Courier Prime)

**Code Style:**
- Readable, well-commented code
- Semantic HTML
- CSS organized by section
- JavaScript functions clearly named

## üö´ What We Won't Accept

- Corporate sponsorships or partnerships
- Removal of leftist framing or working-class perspective
- Attempts to make the tracker "neutral" or "objective"
- Design changes that abandon the protest poster aesthetic
- Contributions from bad-faith actors or right-wing trolls

## ‚úÖ Pull Request Process

1. **Fork the repository**
2. **Create a branch** (`git checkout -b feature/your-feature-name`)
3. **Make your changes**
4. **Test thoroughly** (open index.html in multiple browsers)
5. **Commit with clear messages** (`git commit -m "Add: rent freeze status update"`)
6. **Push to your fork** (`git push origin feature/your-feature-name`)
7. **Open a Pull Request** with:
   - Clear description of changes
   - Why this change is needed
   - Any relevant issue numbers
   - Screenshots (for visual changes)
